1 Qu'est-ce que l'injection SQL ? Comment s'en prémunir ? Une attaque par injection SQL utilise un code SQL malveillant pour manipuler la base de donnée. C'est une action visant un site ou une page précise en utilisant l'utilisateur comme déclencheur, sans qu'il le sache. Pourquoi s'en prémunir on va utiliser les Tokens CSRF ou la demande de confirmation. 4 Définir l'attaque par force brute et l'attaque par dictionnaire Attaque par force brute : L'attaque par force brute est une méthode dans laquelle un attaquant essaie de trouver un mot de passe en essayant toutes les combinaisons possibles de caractères jusqu'à ce qu'il trouve le bon mot de passe. Attaque par dictionnaire : L'attaque par dictionnaire est une méthode dans laquelle un attaquant utilise un dictionnaire ou une liste de mots courants pour essayer de deviner le mot de passe. 5 Existe-t-il d'autres failles de sécurité ? Citer celles-ci et expliquer simplement leur comportement Injection SQL : L'injection SQL est une faille qui permet à un attaquant d'insérer des instructions SQL malveillantes dans une application, généralement via des champs de saisie de données Cross-Site Scripting (XSS) : Le Cross-Site Scripting est une faille qui permet à un attaquant d'insérer des scripts malveillants dans des pages web consultées par d'autres utilisateurs. Cross-Site Request Forgery (CSRF) : Le Cross-Site Request Forgery est une faille qui permet à un attaquant de forcer un utilisateur authentifié à effectuer des actions non voulues sur un site web. Faille de sécurité des sessions : Une faille de sécurité des sessions survient lorsque les mécanismes de gestion des sessions ne sont pas sécurisés Débordement de tampon : Le débordement de tampon est une faille de programmation où une application ne vérifie pas correctement la taille des données entrantes, permettant à un attaquant d'écrire des données malveillantes Inclusion de fichiers arbitraires : Cette faille permet à un attaquant d'inclure des fichiers arbitraires dans une application, ce qui peut entraîner l'exécution de code malveillant ou la divulgation de données sensibles. Faille de gestion des erreurs : Une faille de gestion des erreurs se produit lorsque des messages d'erreur détaillés sont affichés à l'utilisateur final, ce qui peut donner à un attaquant des informations précieuses sur l'infrastructure de l'application et faciliter d'autres attaques. 6 A quoi servent l'authentification et l'autorisation dans un contexte d'application web ? RGPD

Pour plus de réponses checkez la vidéo avec Stéphane sur Teams 1 Qu'est-ce que le RGPD ? Le RGPD c'est Règlement Général sur la Protection des Données 2 Quel est son objectif principal ? Augmenter la protection des utilisateurs en matière de données personnelles ainsi que leurs accès celles-ci 3 Quelle est la date d'entrée en vigueur du RGPD ? Entrée en vigueur du RGPD le 25 mai 2018 4 Quelles sont les sanctions possibles en cas de non-respect du RGPD ? Amende justu'à 20M pour les PME et 4% du CA global du groupe pour les grandes entreprises 5 En France, quel est l'autorité administrative qui s'occupe de faire appliquer le RGPD ? La CNIL = Commission nationale de l'informatique et des libertés 6 Quel est le consentement valide selon le RPGD ? le consentement valide est une autorisation claire, spécifique, informée et librement donnée par une personne pour que ses données personnelles soient collectées, traitées et utilisées par une organisation ou une entité. 7 Qu'est-ce qu'une politique de confidentialité ? une politique de confidentialité est un document juridique qui décrit la manière dont une organisation collecte, utilise, stocke et partage les données personnelles des utilisateurs ou des clients. C'est une déclaration qui informe les individus sur la manière dont leurs informations seront traitées et protégées par l'organisation. 8 Quelle est la durée de conservation maximale des données personnelles selon le RGPD ? Le RGPD ne définit pas la durée précise pendant laquelle les données personnelles doivent être conservées. Le responsable doit identifier et évaluer ses besoins opérationnels, notamment en se référant aux délais de prescription sectoriels afin d'apprécier les durées de conservation au cas par cas. 9 Quels sont les droits des utilisateurs selon le RGPD ? Droit d'accès : Les utilisateurs ont le droit de demander à une organisation de confirmer si elle traite leurs données personnelles et d'obtenir une copie de ces données Droit de rectification : Les utilisateurs ont le droit de demander la correction de leurs données personnelles inexactes ou incomplètes. Droit à l'effacement (ou droit à l'oubli) : Les utilisateurs ont le droit de demander la suppression de leurs données personnelles si elles ne sont plus nécessaires aux fins pour lesquelles elles ont été collectées, si le consentement est retiré ou s'il y a une autre base légale pour l'effacement. Droit de limitation du traitement : Les utilisateurs ont le droit de demander la limitation du traitement de leurs données personnelles dans certaines circonstances, par exemple en cas de contestation de l'exactitude des données ou de contestation du traitement. Droit à la portabilité des données : Les utilisateurs ont le droit de recevoir leurs données personnelles dans un format structuré, couramment utilisé et lisible par machine, et de les transmettre à une autre organisation. Droit d'opposition : Les utilisateurs ont le droit de s'opposer au traitement de leurs données personnelles, notamment pour le marketing direct ou pour des raisons liées à leur situation particulière. Droit de ne pas faire l'objet de décisions automatisées : Les utilisateurs ont le droit de ne pas être soumis à des décisions fondées uniquement sur le traitement automatisé, y compris le profilage, si ces décisions ont des effets juridiques significatifs ou similaires. 10 Qu'est-ce que le principe de minimisation des données selon le RGPD ? le principe de minimisation des données stipule que les organisations ne doivent pas collecter plus de données personnelles que ce qui est réellement nécessaire pour réaliser l'objectif défini
